Inline Elements

The Basics

Inline elements are HTML elements that are laid out horizontally one after the other. They do not begin on a new line by default. Some commonly used inline elements include anchor, img, span, button, input, label and textarea.

Take Note:

  • The width of an inline element is generally determined by its content.
  • An inline element can be contained within a block-level element.
  • An inline element in general cannot contain a block-level element.
  • Inline elements can contain other inline elements like the <a> element that is contained within the <strong> element in the above example.

Things to try:

  • Resize the output area and notice the behavior of the inline elements. The elements should fill horizontally and wrap when the container is not wide enough for the contents.
  • Modify the HTML code so the <strong> element is within the <a> element. Does the link still work? Is the text still bold? While visually it doesn't seem to make a difference, you should keep in mind what you are trying to add strong importance to - the link or the text for the link.

The SPAN element

The <span> element is an inline element without any semantics, other than those that you apply with id or class attributes. It is frequently used to style a group of phrasing elements, but it should not be used when a semantic element (e.g. strong, em, big, small, etc.) would be better suited for the content.

HTML5 and Inline Elements

In HTML5 Element content categories were introduced, which replaced the simple block-level and inline distinction, but the block-level/inline categorization is still used in discussing the default behavior of elements. Mozilla provides a listing of block-level elements that also includes HTML5 elements that are technically not defined within the block-level/inline categorization.

Big Ideas

  • Inline elements are laid out horizontally one after the other.
  • The width of an inline element is generally determined by its content.
  • Inline elements can contain other inline elements.
  • An inline element, in general, cannot contain a block-level element.
  • The span element is an inline element without any semantics that is used to style phrasing elements.
