Box Model

The Basics

In the HTML Overview we saw that every HTML element exists within a rectangle or box on a page. This box has the properties of width, min-width, max-width, height, min-height, max-height, padding, border and margin that can be applied and modified. Understanding the box model, as it is known, is essential to creating layouts that reflect your design. Width and Height

The width and height properties, by default, control the width and height of the content area. The default value for the width and height properties is set to auto, which allows the browser to calculate and select a width/height for the element. Length values for width and height followed by a unit (e.g. em, px, rem) can be used to provide an exact size of the content area. Percentage values can also be used so that the width/height will fill the specified percentage of the containing block's width/height.

The min-width and min-height is used to ensure a minimum width/height of an element and overrides the width/height value. The max-width and max-height are similar, but control the maximum width/height and also override the width/height value. Setting the min-width/height (max-width/height) is often used for cases when a browser is resized and you want to ensure the content area is no smaller (larger) than a particular size.

The width and height properties can be set to be inclusive of the padding and border using the box-sizing property.

Content Area and Edge

The content area contains the actual content - text, image, etc. You can specify the width and height of the content area for block-level elements and replaced inline elements (e.g. images) using CSS, but you cannot specify the width and height for non-replaced elements (e.g. span, strong, em, etc.).

The content edge is not actually a visible edge as illustrated above, but it represents the outer edge of the content area and the inner padding edge, if any padding exists.

Padding Area and Edge

The padding is an optional area between the content and an optional border. Padding is often used to provide "breathing room" between the content and the border for readability and aesthetics.

The padding edge is not actually visible as illustrated above, but it represents the outer edge of the padding area and the inner edge of the border.

Padding can be specified using CSS declarations for each individual side using padding-top, padding-right, padding-bottom, and padding-left.

padding-top: 10px;
padding-right: 10px;
padding-bottom: 10px;
padding-left: 10px;

Shorthand for Padding Declarations

The previous four declarations can also be written with the following short-hand notation.

padding: 10px;

This is the shorthand notation when all the sides have the same padding value. What if you have different values? The shorthand still works, you just need to specify enough information.

Before looking at the other shorthand notation syntax, you should remember the following: Start at the Top and move like a clock. What does this mean? It's hopefully an easy way to remember the order in which the sides are specified for padding shorthand notation. You specify the padding in clockwise order: top, right, bottom and left.

Shorthand for all four sides having different padding values.

padding: top right bottom left; 
padding: 20px 10px 30px 15px;

Shorthand for the left and right sides being the same and the top and bottom values being different. Note the ordering is still clockwise, but the value for the right also specifies the left value when using the three value shorthand notation.

padding: top right/left bottom; 
padding: 20px 10px 15px;

Shorthand for the top and bottom having one value and the right and left sides having another value.

padding: top/bottom right/left;
padding: 20px 10px;

Border Area and Edge

The border is another optional area between the optional padding and margin areas. The border properties width, style, and color can be modified, and each side can have a different border or no border.

The border edge is not actually visible as illustrated, but represents the outer edge of the border, if it exists, and the inner edge of the optional margin.

A solid black border with a width of 2 pixels would be specified as follows: 

border-width: 2px;

style: solid;

color: black;

There is also shorthand notation for specifying the border:

border: border-width border-style border-color; 
border: 2px solid black;

Each side can have its own width, style and color specified using the following syntax.

border-<side>-width: border-width;
border-<side>-style: border-style;
border-<side>-color: border-color;

Where <side> would be replaced by top, right, bottom or left.

Valid values for the properties: border-width, border-style and border-color.

Margin Area and Outer Edge

The margin is an optional area between the optional border and other elements. Margins are what keep elements from "bumping" into one another and are always transparent so that the parent element's background will show through the margin.

The outer margin edge is not actually visible as depicted in the Box Model illustration, but it represents the outer rectangular perimeter of the element on the page. Any area outside of the outer edge would belong to another element.

Margins can be specified using CSS declarations for each individual side much like padding, margin-top, margin-right, margin-bottom, margin-left.

margin-top: 10px;

margin-right: 10px;

margin-bottom: 10px;

margin-left: 10px;

The shorthand notation for margins follows the same notation as padding, but in place of padding in the declarations, you use margin.

Collapsing Margins

In the previous Things to try, when the padding was removed from the div container we could see the light blue background color of the body above the outer edge of the div and paragraph. This is the result of collapsing margins. In the CSS 2.1 specification , top and bottom margins of adjacent and also parent/child block-level elements sometimes combine (collapse) into a single margin that will have the size of the larger of the two collapsed margins.

Here's the quick summary for collapsing margins.

  • Horizontal margins never collapse.
  • The vertical margins of adjacent siblings will collapse, unless a later sibling needs to be cleared (discussed in Float section).
  • The top margin of the first child element and the bottom margin of the last child element (for a single child element, the first and last are the same) will collapse with the parent element's margin except when a border, padding, inline content, or clearing (discussed in Float lesson) separates the two margins. The collapsed margin will exist outside of the parent element.
  • If there is nothing (border, padding, inline content or height) to separate a block element's top and bottom margin, the margins will collapse. This is the case of empty blocks, which are not as common as the previous two cases.

It is the third case that was being exhibited when the padding was removed from the div container in the previous example. The top and bottom margins collapsed and the single collapsed margin became 10px (in this case the margin on the div was 0). The reason the background color of the body was visible, is that the collapsed margin exists outside of the parent element (the div).

Width and Height Calculations and Box-sizing

In creating a website, you will often need to know the width and height that an element will occupy on a page to achieve a desired layout. The default width and height that an element will occupy on a page is calculated by the following:

element width = left margin + left border + left padding + width + right padding + right border + right margin

element height = top margin + top border + top padding + height + bottom padding + bottom border + bottom margin

Often the padding will be added or modified once a layout is defined and text is added. This will usually require adjustments to the width and height, since the width and height, by default, does not include padding. The CSS3 UI specification addresses this issue with the box-sizing property. The box-sizing property allows for the width to include only the content area, the default, or for the width to also include padding or for the width to also include padding and the border.

box-sizing value Description
content-box The width and height only account for the content area. Any padding, border or margin has to be accounted separately when default terming the space that is occupied by an element. This is the default value.
padding-box The width and height include the content area and the padding, but does not include the border or margin. This value has limited very limited browser support.
border-box The width and height include the content area, the padding and the border, but does not include the margin. This value has wide support with modern browsers.

As experienced in the previous Things to try, it is often easier to work with the box-sizing property set to border-box than content-box. The current best practice for applying box-border to all elements is provided below.

html {
  box-sizing: border-box;
}
*, *:before, *:after {
  box-sizing: inherit;
}

Overflow

Sometimes the content of an element will be too large for the specified content area size. So, what happens to the content? Is it chopped off? Does it "overflow" outside of the element? Do you get scrollbars? The answer depends on the value that is set for the overflow property.

Take Note:

The following table summarizes the four overflow values.

Overflow Value Description
visible Allows for the content to visibly overflow outside of the element. This is the default value.
hidden Any content that does not fit within the content area will be clipped.
scroll Scrollbars are added to allow the user to scroll through the content. The scrollbars will be present even if the content fits within the content area.
auto The browser handles overflow where scrollbars will usually be added if the content is larger than the content area.

Which value you set for the overflow property will depend on several factors: the type of content within the element, the adjacent content, the layout, containing elements etc. This lesson will not explore when to use which values, but you can do a web search and find recommendations for setting the overflow property.

Big Ideas

  • Every element on a web page is contained within a "box" that has a height, width, optional padding, optional border and optional margin that can be modified using CSS.
  • Vertical margins can sometimes collapse, but horizontal margins never collapse.
  • The box-sizing property, by default is set to content-box, which specifies that the height and width property will only account for the content area. If the value is set to border-box, then any padding and border will also be included in the height and width.
  • The overflow property can be used to specify what happens to the content when it is too large for the content area.
