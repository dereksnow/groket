Floating Part I

Normal Flow

The CSS layout model specifies that elements are laid out from top to bottom and from left to right by default. This is what we call the normal flow of elements. Let's take a closer look at an example that shows the normal layout of elements.

Take Note:

  • Block and inline elements are laid out top to bottom and from left to right. The block elements (divs) by default occupy the width of the containing element. The inline elements (two imgs and a span) sit adjacent to each other and wrap to the next line only when the width of the container div is exceeded.
  • Each div within the container div has been given a height and a width.
  • There is a space between the images and the div below it. The images sit on the same baseline as the text. The leg of the letter "p" in "Span" sits below the baseline. This space for descenders will exist by default even if there isn't a letter that has a descender.

Things to try:

  • Remove the letter "p" in "Span" to show that the space between the images and the div below it is maintained. This space can be removed by changing the vertical-align (default is baseline) property of the image to top (other values will also work).
  • Change the container div width to 100px and notice how the images and text wrap.
  • Add or remove block and inline elements to see how they are laid out with normal flow.

Floating

The rest of this lesson will look at how we can take elements out of the normal flow of a page using the float property. Elements are not floated by default, but when the value of left or right is assigned to the float property, the element is moved as far left or right as possible. This allows any line boxes that comes after the floated element to wrap around it. A line box is the rectangular area that contains the boxes that form a line. Let's look at an example that illustrates these concepts and the basics of a floated element.

Take Note:

  • The image is the only element that has a non-default float value, in this case left. Since the image is floated, it is initially positioned within the normal flow, but then taken out of the flow and shifted left as far as possible.
  • Since the image is outside of the normal flow, the "box" for the paragraph and div are placed normally.
  • The paragraph has a purple border and fills the width of the containing element, even though the left most edge is hidden by the image.
  • The line box that surrounds the text within the paragraph and div with a blue background has a red dotted border. These line boxes always exist but cannot been seen without attaching span with a border. Notice that the line box is affected by the floated image and wraps around the image, unlike the paragraph block with the purple border or the div.

Things to try:

  • Change the float for the image from left to right and notice the changes. Revert your changes.
  • Add a margin to the img tag of 10px to reveal the left border of the paragraph.
  • Remove the image and notice how the line boxes wrap to fill the containing space.

Inline Text Floated

Floating inline text is performed in the same basic way as we did the image, but a width should always be specified. A width is needed since most browsers will have the text fill the width of the containing element by default. Let's take a look at an example of floating text.

  • A width is specified for the floated span to ensure it doesn't fill the width of the containing element (or collapse to zero width).
  • A top and bottom margin is generated for floated inline text, these top and bottom margins wouldn't exist with non-floated inline text. Floated inline text behaves like a block element.

Things to try:

  • Remove the width declaration to see the span text occupy the width of the containing paragraph.
  • Change the float value from left to right.
  • Add additional inline text elements and float them to the left or right to see the behavior.
